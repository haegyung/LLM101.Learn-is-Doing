# 학생 실습용: Gemini CLI 작업 워크플로

## 목적

이 문서는 학습자가 자신의 실제 작업 1개를 기준으로 `설치 -> 규칙 작성 -> 첫 실행 -> 수정` 흐름을 끝까지 따라가게 하는 기준 튜토리얼이다.

## 시작 전 준비

- 지금 처리할 실제 작업을 한 문장으로 적는다.
- 현재 폴더가 git repo 라면 `git status -sb`로 상태를 본다.
- working tree가 깨끗하고 `origin` 원격이 있을 때만 `git fetch origin`, `git pull --ff-only`를 실행한다.

## 가장 짧은 실행 순서

1. `GEMINI.md`에 작업 목적, 출력 형식, 금지 조건을 적는다.
2. `.gemini/skills/`에 작업을 덜 막히게 하는 작은 skill을 둔다.
3. `.gemini/commands/`에 자주 쓰는 시작/수정 요청을 둔다.
4. 첫 실행을 하고 결과를 `outputs/`에 남긴다.
5. 결과를 읽고 한 번 더 revise 한다.

## 기본 프롬프트

```text
이번 주 실제 작업은 [여기에 작업 1문장].
상황 정보, 선언적 지식, 절차적 지식을 반영해서
지금 단계에서 필요한 첫 산출물을 만들어줘.
```

## PASS 기준

- 실제 작업 1문장이 고정되어 있다.
- 첫 산출물 1개가 `outputs/`에 남아 있다.
- 수정 또는 재실행 1회 흔적이 있다.
