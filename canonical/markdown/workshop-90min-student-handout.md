# Workshop 학생 핸드아웃

## 오늘 할 일

1. 이번 주 실제 작업 1개를 한 문장으로 적는다.
2. Lesson 1 흐름으로 첫 결과를 만든다.
3. 결과를 보고 한 번 더 수정한다.
4. 필요하면 Lesson 2에서 자기 과제에 맞게 remix 한다.

## 가장 짧은 PASS 기준

- `outputs/`에 결과 파일 1개
- 수정 또는 what-i-changed 기록 1개
- 마지막에 다음 행동 1개를 적기

## 막힐 때 먼저 볼 것

- task 문장이 너무 크지 않은지 확인
- git 상태가 꼬였는지 확인
- Lesson 1 README의 시작 요청을 그대로 사용
