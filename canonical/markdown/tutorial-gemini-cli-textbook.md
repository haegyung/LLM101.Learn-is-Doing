# Gemini CLI 실습 교재

## 이 문서의 역할

이 문서는 학생 실습용 workflow를 더 차분하게 설명하는 교재 버전이다.

## 핵심 개념

- `workflow`: 일을 끝내기 위한 작업 순서
- `skill`: 특정 작업을 덜 막히게 해 주는 작은 전문 도우미
- `command`: 자주 쓰는 요청을 바로 실행하는 짧은 호출문
- `preflight`: 시작 전에 먼저 확인하는 점검

## 왜 실제 작업 1문장을 먼저 적는가

작업 문장이 먼저 있어야 모델이 일반론이 아니라 현재 과제를 기준으로 움직인다.

## 최소 성공 구조

1. 작업 1문장
2. 규칙 문서 1개
3. 시작 실행 1회
4. 결과 파일 1개
5. 수정 흔적 1회
